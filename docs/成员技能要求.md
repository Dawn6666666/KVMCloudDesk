# KVM 虚拟化管理系统 — 小组成员技能与技术栈要求

本项目是一个采用前后端分离架构的云平台虚拟机管理系统。为了确保项目的顺利推进以及各模块的高质量交付，本项目对三位团队成员提出了相应的专业技能和技术栈要求。以下是详细的成员技能指南。

---

## 成员 A（后端核心开发负责人）

后端核心开发负责人需要精通 Java 开发与底层集成技术，掌握 KVM 虚拟化底层原理，能够处理堆外内存与 C 语言库的动态加载与安全调用。

### 1. 核心编程语言与框架（Java 21 与 Spring Boot 3）
*   **Java 21 新特性**：熟练运用 `Record`、`Switch 模式匹配` 等现代 Java 语法，熟练进行并发编程。
*   **Spring Boot 3.3.6**：
    *   **核心注解与控制流**：掌握 IoC 容器、AOP 编程，熟练运用 `@Profile` 进行开发环境和生产部署环境的条件装配与 Bean 隔离。
    *   **统一异常与响应**：能够设计基于 `@RestControllerAdvice` 和 `@ExceptionHandler` 的全局异常拦截机制，规范统一数据返回格式。
    *   **静态路由转发**：熟练编写 `WebMvcConfigurer`，处理前端 History 路由的单页应用刷新 404 问题。

### 2. 底层 C 语言库桥接（JNA 桥接）
*   **指针与内存管理**：理解 Java 堆内存与物理堆外内存的区别。在完成虚拟机、存储池、网络、快照等底层操作后，必须通过 JNA 显式释放 C 指针，杜绝堆外内存泄漏。
*   **数据结构映射**：熟练定义 JNA 映射接口，继承 `Library`。能将 C 语言中的结构体映射为 Java 中的 `Structure` 派生类。
*   **动态库加载**：知晓如何在不同运行环境下动态加载指定物理路径的动态链接库并优雅降级。

### 3. KVM 虚拟化与 libvirt 核心 API
*   **Libvirt 核心接口**：理解并熟练调用连接管理、虚拟机生命周期、虚拟网络、存储池与存储卷、快照等底层核心 API。
*   **XML 生成与安全防御**：掌握 XML 描述文件的生成，在解析时显式禁用外部 DTD 声明与实体引用展开，防范安全漏洞。
*   **VNC 代理网关**：熟练掌握 WebSocket 协议原理，能够编写 VNC 代理类以建立双向流代理，实现远程桌面的数据中转。

---

## 成员 B（Web前端开发负责人）

Web前端开发负责人需要熟练掌握现代前端工程化开发，能够精细化处理复杂的生命周期时序与实时数据看板绘制。

### 1. 核心技术选型（Vue 3 与 TypeScript、Vite）
*   **Vue 3**：掌握组件化开发模式，熟练运用组合式 API 及各种生命周期钩子，确保数据交互时的高响应性。
*   **TypeScript 类型校验**：在前端编写与后端数据传输对象严格对应的类型定义文件，保证字段结构完全对齐。
*   **Vite 构建工具**：熟悉前端工程脚手架的配置，配置开发代理服务器，支持在本地开发调试时将请求转发到后端。

### 2. 全局状态、日志与网络请求（Axios 与 Pinia）
*   **Axios 拦截器**：封装统一的请求库，编写全局请求与响应拦截器。在每次请求发出时通过请求头携带操作描述，并在响应收到后根据状态码执行统一报错与日志上报。
*   **Pinia 状态管理**：设计并维护全局状态，缓存操作日志并提供实时滚动的日志看板。

### 3. 数据可视化与远程桌面控制（ECharts 与 noVNC）
*   **ECharts 动态监控**：掌握 ECharts 折线图、饼图、环形图的动态数据渲染，展示物理负载和网络吞吐速率的历史趋势。
*   **noVNC 集成**：使用 HTML5 Canvas 容器集成 noVNC 客户端，通过 WebSocket 连接到后端，实现虚拟机直连控制台。

## 成员 C（部署测试与文档编写负责人）

部署测试与文档编写负责人需要具备扎实的系统运维、网络配置与质量保障能力，是系统能够从代码变成可用服务并顺利结题的关键保证。

### 1. Linux 系统运维与虚拟化环境管理
*   **CentOS 环境管理**：熟练掌握 Linux 命令行操作，掌握软件安装与服务配置，熟悉 systemd 服务管理与运行问题排查。
*   **KVM 虚拟化与系统检测**：熟练通过硬件虚拟化标志、内核模块以及设备权限排查系统的虚拟化环境健康度。
*   **宿主机网络与安全机制**：
    *   熟练配置桥接网络、NAT 虚拟网络。
    *   精通防火墙配置，管理端口策略以允许局域网访问。
    *   熟练排查由本地主机名反向解析缺失导致的连接延迟问题。

### 2. 系统部署与反向代理（Nginx 与 Spring Boot 打包）
*   **多环境部署拓扑**：理解前后端分离系统的生产部署方案：
    *   **内嵌静态资源部署**：掌握利用前端打包产物拷贝到后端静态资源目录下，通过 Maven 打包构建出一体化 Jar 的方案。
    *   **独立反代部署**：精通 Nginx 的静态资源托管与反向代理配置，重点掌握如何开启 WebSocket 协议升级配置以支持 VNC 连接。

### 3. 质量保障与技术文档编写（UML 与 Markdown）
*   **联调测试与用例设计**：参与前后端系统的功能联调，理解虚拟机的生命周期状态机模型，设计并执行功能测试用例。
*   **UML 建模与图表绘制**：熟练运用图表工具，绘制系统的部署架构图、用例图、状态流转图以及时序图。
*   **技术文档输出**：精通 Markdown 排版，主导最终交付文档的整理与正式课程设计报告的撰写。

## 团队协作与交付规范

为了保障三位成员的成果能够融合，团队需要遵守以下协作共识：
1. **数据契约对齐**：后端编写完数据实体后，应第一时间通知前端进行类型更新，防止数据解析失败。
2. **测试与排错协作**：测试或部署中发现报错时，各成员需配合检查主机日志、后端日志与前端请求路径。
3. **甘特图与里程碑**：小组成员需遵循规划的进度时间表，以流程化方式有序交付成果。
